## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, minimal padding |
| sm | 8px | Compact element spacing (e.g., between subtitle and title on spine) |
| md | 16px | Default field spacing in add-book form |
| lg | 24px | Section padding (e.g., between form groups) |
| xl | 32px | Layout gaps, page horizontal padding |
| 2xl | 48px | Major section breaks, top-level padding |

Exceptions:
- **Shelf row height:** TBD during planning (approximately 200–240 pixels based on portrait orientation)
- **Spine width:** Fixed width for all spines (TBD during planning, ~50–60 pixels for 6–8 books per row on standard mobile)
- **Touch targets (icon buttons):** 44px minimum to meet accessibility standards

## Visual Specifications

### Shelf Design (D-01 to D-05)

**Shelf Container:**
- Background: Warm brown gradient (exact colors TBD during planning; must feel like illustrated wood, not photograph)
- Thick dark horizontal lip at bottom: ~8–12px height, color ~black54 or charcoal
- Rounded corners: BorderRadius.zero (sharp corners, befitting old-money aesthetic)
- Elevation/shadow: Subtle drop shadow to lift shelf off parchment background (TBD)

**Spine Layout:**
- Fixed spine width: TBD during planning (~50–60px for 6–8 books per row)
- Spines fill row horizontally, wrap to new shelf when full
- Approximately 6–8 books per row (based on standard mobile portrait width ~375px)
- Text rotation: 90° clockwise (text reads vertically down spine, like real books)

**Spine Content:**
- Title (primary): Rotated 90°, centered vertically on spine, weight 400, size 14–16px (TBD)
- Author (secondary): Smaller text below title (size ~12px, TBD), also rotated, lighter color (Colors.black54)
- Cover colour background: Solid colour from user-selected swatch

**Empty State:**
- Single wooden shelf rendered (even with no books)
- Centered italic prompt: "Add your first book" (size 14px, Georgia, Colors.black54)
- Shelf is still visible to reinforce metaphor

### Add Book Flow (D-06 to D-09)

**Entry Point:**
- Floating action button removed (not using FAB)
- App bar "+" button (icon button, no label) in top-right corner
- Tapping navigates to `/add-book` route (GoRouter, full-screen presentation)

**AddBookScreen Layout:**
- Title at top: "Add Book" (size 32px, weight bold, Georgia)
- Single-column form with fields stacked vertically
- Form fields:
  - Title (TextField, label "Title", white background, 2px black87 border, BorderRadius.zero)
  - Author (TextField, label "Author", white background, 2px black87 border, BorderRadius.zero)
  - Cover Colour (8-swatch colour picker, displayed as tappable square swatches, ~50x50px each, with border on selected swatch)
  - Notes/Review (TextField, label "Notes (optional)", multiline, white background, 2px black87 border, BorderRadius.zero)
- Buttons at bottom:
  - Save (ElevatedButton, backgroundColor forestGreen, white text, "Save", 2px black87 border, BorderRadius.zero, height 52px)
  - Back/Cancel: Standard platform back navigation (GoRouter pop)

**Form validation:** TBD during planning (placeholder: required fields are Title + Author; Cover has default selection)

**Return behavior:**
- On save: return to LibraryScreen (GoRouter pop)
- Newly added book appears immediately on shelf (assumes real-time Supabase subscription or immediate fetch)

**No image upload in v1:** Cover is colour palette only (D-09 deferred)

### Drag Reorder (D-10 to D-12)

**Interaction:**
- User long-presses or swipes a spine to initiate drag
- Drag reorder package TBD during planning (flutter_reorderable_gridview, ReorderableListView, or custom)
- Visual feedback during drag: TBD (e.g., elevated spine, opacity change, placeholder space)
- On drop: Update position column in Supabase books table for affected books

**Persistence:**
- Position updates immediately reflected on screen
- Persists after app close/reopen (reads from Supabase books table on init)

## Accessibility Requirements

- Icon button size: minimum 44x44px touch target
- Text contrast: 4.5:1 for body text on parchment background (parchment #F5F0E8 on black87 text meets WCAG AA)
- Form labels: associated with inputs via TextField label property
- Error messages: displayed in red (#8B1A1A) with explicit text, not icon-only
- Colour picker: tappable swatches must be large enough (50x50px minimum) and have focus/selected states visible beyond colour alone
