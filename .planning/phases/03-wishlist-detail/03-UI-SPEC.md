| Screen | Heading | Body |
|--------|---------|------|

**Placement:** Center of first wooden shelf widget, same layout as LibraryScreen empty state.

### Error States

| Condition | Message | Action |
|-----------|---------|--------|
| Load wishlist fails | "Error loading wishlist. Please try again." | Show in Center widget; include a manual refresh option (icon button) |
| Move to library fails | "Error moving book to library. Please try again." | Show as error style text below button; button remains enabled for retry |
| Network error on detail view | "Error loading book details. Please try again." | Show in Center widget with retry affordance |

### Destructive Actions

**Move to Library confirmation:**
- No modal dialog required — button action is clear from context (wishlist book → library)
- Post-action navigation: `context.pop()` returns to WishlistScreen (not LibraryScreen)
- Toast/snackbar (optional): "Book moved to library" confirmation

**No other destructive actions in Phase 3.**

## Layout Patterns

### WishlistScreen

- **App bar:** Title "Wishlist" (Display style, 48px bold), '+' button (visible to her only), logout button
- **Body:** SingleChildScrollView → ReorderableWrap (disabled for wishlist) → ShelfWidget list
- **Empty state:** Centered empty shelf with muted "Her wishlist is empty" prompt
- **Loading:** Centered CircularProgressIndicator (forestGreen color)
- **Error:** Centered error message with retry icon button

### BookDetailScreen

- **App bar:** Back button, book title (optional display), action menu (optional; not required for Phase 3)
- **Body:** SingleChildScrollView
  - Cover swatch: 120×160px centered, solid color block (not an image in Phase 3)
  - Title: Heading style (32px bold), 16px padding top/bottom
  - Author: Body style (16px), 16px padding bottom
  - Notes section (if present): "Notes" label (14px), 8px gap, notes body text (16px), 16px padding bottom
  - Notes section (if null): hidden entirely
  - Whitespace gap: 24px (lg)
  - "Move to Library" button (wishlist only): full width, 16px horizontal margin, 24px bottom margin
  - If library book: button hidden, whitespace removed
- **Bottom padding:** 24px safe area for iOS home indicator

### Navigation

- **WishlistScreen → BookDetailScreen:** `context.push('/book-detail', extra: BookModel)`
- **LibraryScreen → BookDetailScreen:** `context.push('/book-detail', extra: BookModel)` (same route, context-aware)
- **BookDetailScreen → back:** `context.pop()` returns to originating screen (wishlist or library)
- **After move-to-library:** `context.pop()` returns to WishlistScreen (intent: remove from wishlist view immediately)

## State & Interaction Specifics

### Loading States

- **Wishlist load:** CircularProgressIndicator (forestGreen) while fetching wishlist books
- **Detail load:** CircularProgressIndicator while fetching book details (if async)
- **Move action:** Button shows loading state (disabled + progress indicator inside button, or text → "Moving...")

### Optimistic UI

- **Move to library:** Immediate `context.pop()` without await (optimistic — assume success, handle error async if needed)
- **Alternative:** Wait for Supabase response, then `context.pop()` (safer, ~500ms delay acceptable)
- **Decision:** Use alternative (wait for response) to match Phase 2's `updatePositions` async pattern

### Tap/Interaction Targets

- Book spine: Tappable area (full spine widget, ~60×160px minimum)
- '+' button: 48×48px icon button (Material standard)
- "Move to Library" button: full-width, 48px minimum height
- Back button: 48×48px icon button

---

## Form Validation & Error Handling

### Wishlist Add (new book)

- **Route:** Use existing `/add-book` route with query param `isWishlist=true` OR separate `/add-wishlist` route
- **Decision:** Reuse `/add-book` with optional `isWishlist: true` parameter passed via GoRouter `extra`
- **Validation:** Same as Phase 2 (title + author required, notes optional)
- **Post-add:** Navigate back to WishlistScreen, refresh list

### Move to Library Validation

- No client-side validation needed (book object already validated)
- Server-side: Supabase RLS ensures only books owned by the user (UID) can be modified
- Post-move: Set `is_wishlist = false`, assign next position, refresh UI
