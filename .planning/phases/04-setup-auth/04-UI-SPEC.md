## Form Component Specifications

### Login Form Layout

**Container:**
- Background: white (derived from Secondary #FFFFFF as fill)
- No explicit card border in Phase 4 (form is inline on parchment background)
- Padding: horizontal xl (32px), vertical 2xl (48px)
- Max width: not constrained (centered by viewport, scrolls on small screens)

**Form Fields:**
- All inputs: height 52px (exact specification from Flutter equivalent)
- Border: 2px solid #222222 (black87), no border-radius
- Focused border: 2px solid #2D4A3E (forest green accent)
- Fill: white
- Label: Georgia serif, 16px, 400, positioned above input (labelText in Material/HTML)
- Text: Georgia serif, 16px, 400
- Padding: 8px (label), TBD internal field padding (executor to match Flutter visual)

**Error Message:**
- Text color: #8B1A1A (dark red)
- Font: Georgia serif, 13px, 400
- Position: below password field, above button
- Padding: top 4px, bottom 4px
- Visibility: conditional (shown only when `_errorMessage` is not null)
- Message text: "Invalid email or password" (generic, no field-specific errors)

**Button:**
- Element: "SIGN IN"
- Background: #2D4A3E (forest green)
- Text color: white
- Font: Georgia serif, 16px, bold (700), letter-spacing 1px
- Border: 2px solid #222222 (black87)
- Height: 52px
- Width: 100% (full form width)
- Border-radius: none (0)
- On hover: TBD by executor (no spec in Flutter, recommend darkened forest green or visual emphasis)
- On disabled (loading state): opacity maintained, cursor not-allowed
- On click (loading): button text replaced with circular progress spinner (20x20px, white, 2px stroke width)

## Route Protection & Interaction States

### Public Routes
- `/login` — Login page (accessible to all)

### Protected Routes
- All routes except `/login` require authentication
- Unauthenticated access redirects to `/login` (middleware-enforced)

### Session Persistence
- Supabase session stored in browser (via @supabase/ssr)
- Page refresh: session persists (user remains logged in)
- Browser tab close: session persists on reopen (stored in localStorage or HTTP-only cookie per @supabase/ssr)

### Button States

| State | Button | Text/Content |
|-------|--------|--------------|
| Idle | enabled, clickable, forest green background | SIGN IN |
| Loading (during sign-in) | disabled, not clickable, forest green background | Circular spinner (20x20px white) |
| Error | enabled, clickable, forest green background | SIGN IN (error message shown separately above) |
