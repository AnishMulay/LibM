## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #F5F0E8 (parchment) | Page background, surfaces |
| Secondary (30%) | #2D4A3E (forest-green), #8B1A1A (dark-red) | Shelf background, book spines |
| Accent (10%) | #000000 (border) | Shelf lip, spine borders, typography emphasis |
| Destructive | #8B1A1A (dark-red) | Error states only |

**Accent reserved for:** Shelf lip (12px+ bottom border), spine borders (when needed), text contrast emphasis.

**Wood Tones (NEW — Phase 5):** Add three new color tokens to `web/tailwind.config.ts` and mirror in `web/app/globals.css`:

| Token | Value | Usage |
|-------|-------|-------|
| wood-light | #8B6F47 | Top of shelf gradient |
| wood-mid | #6B5438 | Mid shelf gradient |
| wood-dark | #4A3728 | Bottom shelf gradient / shelf lip base |

**Wood Gradient Construction:** Linear gradient from `wood-light` (top) → `wood-mid` (middle) → `wood-dark` (bottom) over the shelf height. The shelf lip (bottom border) should use `wood-dark` or darker for strong visual separation.

**Auto-Contrast Text (D-07):**
- **White text (rgb 255, 255, 255)** on dark book spine colors: forest-green (#2D4A3E), dark-red (#8B1A1A), charcoal, navy, burgundy
- **Black text (rgb 0, 0, 0)** on light book spine colors: parchment (#F5F0E8), cream, aged gold
- **Computation:** Use WCAG luminance formula at render time: if `(R*0.299 + G*0.587 + B*0.114) / 255 < 0.5`, use white; else use black.

**Source:** D-01 (warm brown gradient), D-07 (auto-contrast text), and Phase 4 color tokens.

## Component Specifications

### Bookshelf Component

**File:** `web/components/bookshelf/Bookshelf.tsx`

**Responsibility:** Render one or more horizontal shelf rows filled with BookSpine components.

**Props (Claude's Discretion — design for Phase 6 integration):**

Choose one pattern:
- **Pattern A (children):** `<Bookshelf><BookSpine ... /><BookSpine ... /></Bookshelf>` — children are BookSpine elements
- **Pattern B (books array):** `<Bookshelf books={[...]} />` — array of book objects

**Recommended:** Pattern B (books array) for simpler Phase 6 integration with Supabase data fetching.

**Visual Specification:**

- **Shelf height:** Approximately 140–160px per row (Claude discretion; must fit title + author text without truncation)
- **Background:** Linear gradient from `wood-light` → `wood-mid` → `wood-dark` (top to bottom)
- **Bottom border (shelf lip):** 12px or thicker, solid `wood-dark` color; intentionally heavy for strong visual divider
- **Shelf row gap:** 8px vertical space between rows (`gap-sm`)
- **Spine container:** Flex row with horizontal wrapping; spines flow left-to-right and wrap to next row when full
- **Row justification:** Flex-start (spines align left within each row)

**Accessibility:**
- Semantic `<div role="region" aria-label="Bookshelf">` or `<section>`
- No interactive behavior at Bookshelf level (Phase 5 is static); Phase 6 adds drag handling

### BookSpine Component

**File:** `web/components/bookshelf/BookSpine.tsx`

**Responsibility:** Render a single book spine with rotated title and author text in the book's cover color.

**Props (Claude's Discretion — design for Phase 6 integration):**

```typescript
interface BookSpineProps {
  id?: string;              // For phase 6 drag-and-drop keys
  title: string;            // Book title, rotated 90° down the spine
  author: string;           // Author name, rotated, smaller
  coverColor: string;       // Hex or CSS color name (e.g., '#2D4A3E')
  onTap?: () => void;       // Optional click handler (for phase 6)
}
```

**Visual Specification:**

- **Width:** ~48–64px fixed (Claude discretion; target 6–8 spines per row at typical 1280px viewport width)
- **Height:** Inherited from Bookshelf row height (~140–160px)
- **Background:** Solid `coverColor` (e.g., forest-green, dark-red, parchment, cream, etc.)
- **Border:** 2px solid black (Tailwind `border-2 border-black`)
- **Text Layout:**
  - Title: rotated 90° (CSS `writing-mode: vertical-rl` or `transform: rotate(-90deg)`)
  - Author: rotated 90°, smaller font size (Subtitle or smaller), positioned below title within spine
  - Text color: Auto-contrast computed at render (white or black per luminance)
  - Text padding: 4–6px from spine edges
  - Text truncation: Ellipsis or fade if title/author exceed spine width (Claude discretion)

**Text Rendering Details:**
- **Title:** Body font (16px), weight 400, rotated vertically
- **Author:** Subtitle font (14px), weight 400, rotated vertically, positioned below title
- **Vertical centering (within rotated space):** Text should center visually on the spine

**No interactive behavior in Phase 5:** Click handlers are defined in prop interface but not wired until Phase 6.
