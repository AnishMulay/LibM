## Spacing Scale

| Token | Value | Usage |

## Color

Inherited from Phase 4–6 (no changes required for Phase 7):

| Role | Hex | Usage |
|------|-----|-------|
| **Dominant (60%)** | `#F5F0E8` (Parchment) | Background, page surfaces, input backgrounds, card backgrounds |
| **Secondary (30%)** | `#8B6F47` / `#6B5438` / `#4A3728` (Wood tones) | Bookshelf background gradient (linear-gradient to bottom) |
| **Text Primary (10%)** | `#222222` (Charcoal) | Body text, headings, form labels |
| **Text Secondary** | `#666666` (Gray) | Author text, helper labels, empty state secondary copy |
| **Accent** | `#D4AF37` (Aged Gold) | Input focus borders, selected color swatch borders, primary button fill |
| **Borders** | `#000000` (Black) | 2px borders on all interactive elements, 14px bookshelf bottom border |
| **Destructive** | `#8B1A1A` (Burgundy) | Validation error borders, error text color |

**Accent reserved for:**
- Input/textarea focus border (2px solid #D4AF37)
- Selected color swatch border outline
- Primary button background (aged-gold fill)
- BookSpine contrast text calculation (WCAG luminance-based: white on dark, black on light)

**Bookshelf wood gradient:** linear-gradient(to bottom, #8B6F47 0%, #6B5438 50%, #4A3728 100%) + 14px solid #4A3728 bottom border.

**Phase 7 color contract:**
- Wishlist page background: inherit parchment from global
- "Move to Library" button: 2px black border, no border-radius, forest-green (#2D4A3E) OR aged-gold (#D4AF37) fill (executor discretion; recommend forest-green for semantic distinction from add/save buttons)
- Back link color: #2C2C2C (existing pattern from `/books/[id]`)

### Interaction States

| State | Behavior | Feedback |
|-------|----------|----------|
| Input focus | User taps/clicks text input or textarea | Border color changes from #000000 to #D4AF37 (aged gold) |
| Input blur | User leaves input field | Border color reverts to #000000 (or #8B1A1A if error) |
| Color swatch selected | User clicks a color swatch | 2px solid #D4AF37 border + 2px solid #D4AF37 outline with 2px offset |
| Button disabled | Form is saving or button is disabled | Opacity 0.5, cursor: not-allowed |
| Move to Library in progress | User taps "Move to Library" button | Button text changes to "Moving...", button disabled (cursor: not-allowed, opacity 0.5), wishlist shelf remains visible during request |
| Move to Library success | Move completes without error | Navigate to `/wishlist` (user sees fresh wishlist without the moved book) |
| Move to Library error | Move fails (network, Supabase error) | Show alert "Failed to move. Please try again." below button in burgundy #8B1A1A; button re-enabled |

**No confirmation dialog for Move to Library.** One-tap only (per D-11).

## Form Layout & Structure

### Wishlist Add Form (`/wishlist/add`)

**Identical to `/library/add` form structure:**

1. Page heading: "Add to Wishlist" (48px display)
2. Container: max-width 600px
3. Title field: label + input (52px height, 16px text)
4. Author field: label + input (52px height, 16px text)
5. Cover color picker: label + 4-column swatch grid (8px gap)
6. Notes textarea: label + textarea (4 rows default, resizable, 16px text)
7. Button group: Save Book (primary) + Cancel (secondary), side-by-side, flex row, 16px gap

**Validation:**
- Validate on form submit only (not on blur)
- Show field-level errors in burgundy below the field
- Show Supabase errors as an alert role above the form

**Navigation:**
- Cancel: navigate to `/wishlist`
- Save success: navigate to `/wishlist` (SSR re-fetch shows new book)

### Book Detail Page (`/books/[id]`) — Layout

**Existing structure (Phase 6):**

1. Container: min-height screen, parchment background
2. Main div: px-md py-lg (16px horizontal, 24px vertical)
3. Back link: "← Library", 14px label, inline-block, 32px margin-bottom
4. Info container: max-width 600px, parchment bg, 2px black border, 32px padding
5. Title: 48px display, 8px margin-bottom
6. Author: 14px label, 32px margin-bottom
7. Cover swatch: 120×160px, 2px black border, 32px margin-bottom
8. Notes section: heading (14px label) + body (14px, charcoal)

**Phase 7 extension (when `isWishlist === true`):**

- Replace back link text: "← Library" becomes "← Wishlist" (same styling, different href)
- After notes section: 32px margin-top, add Move to Library button
- Button: 52px height, 2px black border, no border-radius, forest-green OR aged-gold fill, 16px text, Georgia serif, charcoal text color

## Bookshelf Component Reuse

### Wishlist Shelf (`/wishlist`)

| Aspect | Contract |
|--------|----------|
| Component | `Bookshelf.tsx` (static, no drag) |
| Client wrapper | Thin `'use client'` wrapper component (new). Name TBD by executor (e.g., `WishlistShelf.tsx` or `ClientWishlistShelf.tsx`) |
| Props | `books: BookSpineProps[]` (same as library shelf) |
| Styling | Inherit Bookshelf gradient, borders, empty state |
| Interactivity | BookSpine `onTap` wired to `router.push('/books/' + bookId)` in wrapper |
| Empty state | Inherited from Bookshelf.tsx: "Add your first book" heading + "No books yet..." body (14px label weight, text-secondary) |
| No drag | Static shelf; dnd-kit NOT required for Phase 7 |

**Server Component (`/wishlist/page.tsx`):**
- Fetch `is_wishlist = true` books, ordered by `created_at` ascending
- Call `supabase.auth.getUser()`, compare `user.id === process.env.NEXT_PUBLIC_HER_UID`
- If match: render "+" add button (48×48px, aged-gold, no label)
- If no match: render nothing (silent asymmetry)
- Pass books to client wrapper

**Client wrapper:**
- Accept `books` and `showAddButton` as props
- Wrap `Bookshelf` component
- On BookSpine tap: `router.push('/books/' + bookId)`
